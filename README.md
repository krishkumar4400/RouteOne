# 🚚 Lanecast

## “Never Let a Truck Run Empty”

---

## 📌 Overview

Lanecast is a logistics coordination platform designed to eliminate empty truck returns by creating a real-time, high-trust matching system between **truck owners (supply)** and **shippers (demand)**.

Unlike traditional freight marketplaces, Lanecast focuses on:

- Guaranteed return loads
- Faster matching
- Reduced dependency on brokers
- Increasing truck utilization

---

## 🎯 Problem Statement

India’s logistics ecosystem suffers from severe inefficiencies:

- 30–40% trucks run empty on return trips
- Matching of trucks and loads is manual (calls, brokers, WhatsApp)
- No real-time visibility of supply and demand
- Heavy reliance on intermediaries (brokers take 10–20% cut)
- Small fleet owners (1–10 trucks) dominate → highly fragmented market

### 💣 Core Problem

> There is no real-time coordination layer between fragmented supply (trucks) and fragmented demand (shipments).

---

## 💡 Solution

Lanecast provides a **controlled logistics network** that:

- Ensures trucks always get a return load
- Matches shipments with trucks in near real-time
- Reduces idle time and empty runs
- Builds trust through predictable pricing and payments

## 🔄 Product Flow

---

### 📦 Shipper Flow (Create Shipment)

1. Login / Signup
2. Create Shipment:
   - Pickup location
   - Drop location
   - Load details (weight/type)
   - Date & time
3. System suggests:
   - Price
   - Available trucks
4. Confirm booking
5. Track shipment in real-time
6. Delivery confirmation
7. Payment processed

---

### 🚚 Truck Owner Flow

1. Login / Signup
2. Add truck details
3. Set availability status:
   - Available
   - On trip
4. When idle:
   - Receive load suggestions
5. Accept / Reject load
6. Complete delivery
7. Get payment

## 5. DATABASE DESIGN

### 1. Users Table

```sql
CREATE TABLE users (
    id UUID PRIMARY KEY,
    name VARCHAR(100),
    phone VARCHAR(15) UNIQUE NOT NULL,
    role VARCHAR(20) CHECK (role IN ('shipper', 'fleet_owner', 'driver')),
    rating FLOAT DEFAULT 5.0,
    is_verified BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP DEFAULT NOW()
);
```

### 2. TRUCKS TABLE

```sql
CREATE TABLE trucks (
    id UUID PRIMARY KEY,
    owner_id UUID REFERENCES users(id),
    truck_number VARCHAR(20) UNIQUE,
    type VARCHAR(50),
    capacity_kg INT,
    current_location GEOGRAPHY(POINT, 4326),
    status VARCHAR(20) CHECK (status IN ('available', 'on_trip', 'offline')),
    last_updated TIMESTAMP,
    created_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_truck_location ON trucks USING GIST (current_location);
```

### 3. LOADS TABLE

```sql
CREATE TABLE loads (
    id UUID PRIMARY KEY,
    shipper_id UUID REFERENCES users(id),
    pickup_location GEOGRAPHY(POINT, 4326),
    drop_location GEOGRAPHY(POINT, 4326),
    weight_kg INT,
    price NUMERIC,
    status VARCHAR(20) CHECK (
        status IN ('pending', 'assigned', 'in_transit', 'completed', 'cancelled')
    ),
    scheduled_at TIMESTAMP,
    created_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_load_pickup ON loads USING GIST (pickup_location);
```

### 4. TRIPS TABLE

```sql
CREATE TABLE trips (
    id UUID PRIMARY KEY,
    truck_id UUID REFERENCES trucks(id),
    load_id UUID REFERENCES loads(id),
    driver_id UUID REFERENCES users(id),
    status VARCHAR(20) CHECK (
        status IN ('assigned', 'started', 'completed', 'cancelled')
    ),
    start_time TIMESTAMP,
    end_time TIMESTAMP,
    created_at TIMESTAMP DEFAULT NOW()
);
```

### 5. PAYMENTS TABLE

```sql
CREATE TABLE payments (
    id UUID PRIMARY KEY,
    trip_id UUID REFERENCES trips(id),
    amount NUMERIC,
    platform_fee NUMERIC,
    payout_amount NUMERIC,
    status VARCHAR(20) CHECK (
        status IN ('pending', 'completed', 'failed')
    ),
    created_at TIMESTAMP DEFAULT NOW()
);
```

### 6. RATINGS TABLE

```sql
CREATE TABLE ratings (
    id UUID PRIMARY KEY,
    trip_id UUID REFERENCES trips(id),
    from_user UUID REFERENCES users(id),
    to_user UUID REFERENCES users(id),
    rating INT CHECK (rating BETWEEN 1 AND 5),
    review TEXT,
    created_at TIMESTAMP DEFAULT NOW()
);
```

### 7. LOCATION TRACKING

```sql
CREATE TABLE truck_locations (
    id BIGSERIAL PRIMARY KEY,
    truck_id UUID,
    location GEOGRAPHY(POINT, 4326),
    recorded_at TIMESTAMP DEFAULT NOW()
);
```
